# Project Rules

## Scope
Only work in this folder and its subfolders:

`/Users/brimay/Documents/Bri May/Github/bri-portfolio-2`

Do not create, edit, or reference files outside this directory.

## Platform
This site must work on GitHub Pages.

Use only:
- HTML
- CSS
- Vanilla JavaScript

Do not use:
- Backends
- Node servers
- Build tools
- Frameworks requiring compilation

Everything should run as a static site.

## Workflow
- Review existing files before making changes
- Extend existing code instead of rewriting unless asked
- State which files will change before editing
- Output full file contents with file paths when creating/editing files

## Structure
Prefer this structure:

- `/index.html`
- `/work/`
- `/about/`
- `/assets/`
- `/styles/`
- `/scripts/`

Avoid duplicate files and unnecessary folders.

## Design System
Use a minimal, high-end visual style inspired by Apple / Stripe / Airbnb.

### Color
- Background: `#FFFFFF` or `#FAFAFA`
- Text: `#111111`
- Muted text: `#666666` to `#888888`
- Border: `#EAEAEA`
- One accent color maximum, used sparingly

### Typography
- Inter or system sans-serif
- Large bold headings
- Clear hierarchy
- Readable body text

### Spacing
Use a consistent spacing scale:
- `16px`
- `24px`
- `32px`
- `48px`
- `80px`
- `120px`

### Layout
- Max width: `1200px`
- Centered content
- Generous whitespace

## Components

### Hero
- Left: text
- Right: image
- Strong headline
- Minimal supporting copy
- Optional subtle parallax only

### Sections
Use this pattern:
- optional label
- title
- short supporting text
- content block

### Cards
- Border: `1px solid #EAEAEA`
- Radius: `12px`
- Padding: `24px`
- Subtle hover lift

### Buttons
- Primary: dark background, white text
- Secondary: minimal / text / outline

### Images
- Rounded corners
- High quality
- No heavy shadows
- Very subtle hover zoom max

### Motion
- Fade + slight upward reveal only
- No flashy animations

## Case Study Structure
Default order:
1. Hero
2. Overview
3. Problem
4. Opportunity
5. Constraints
6. Approach
7. Key Decisions
8. Experience
9. Outcomes
10. What’s Next
11. Final Takeaway

Keep case studies highly scannable. Avoid long text blocks.

## Consistency
Reuse existing styles and components whenever possible.
Do not invent a new visual system unless explicitly asked.

## Goal
Build a clean, cohesive, senior-level portfolio that feels intentional, modern, and fully deployable on GitHub Pages.
